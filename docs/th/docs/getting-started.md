## สิ่งที่คุณจะทำเสร็จภายใน 10 นาที

- ตั้งค่าสิทธิ์
- สกรีนช็อตและมาร์กอัปครั้งแรก
- หน้าต่างปักหมุดครั้งแรก
- ตรวจสอบฟีเจอร์เฉพาะแพลตฟอร์ม
- ตรวจสอบการชนกันของปุ่มลัด

## ข้อกำหนดเบื้องต้นของระบบ

### macOS

- macOS 14+
- สิทธิ์บันทึกหน้าจอ
- สิทธิ์การช่วยสำหรับผู้พิการ แนะนำสำหรับการตรวจจับหน้าต่างและเวิร์กโฟลว์ที่ราบรื่นยิ่งขึ้น

### Windows

- Windows 10+
- สิทธิ์ไมโครโฟนเฉพาะเมื่อบันทึกเสียง

### Linux beta

SnapTray มีให้ในรูปแบบ AppImage สำหรับ Ubuntu 22.04 X11 beta การบันทึกหน้าจอและ OCR ไม่รวมอยู่ใน Linux beta และ UI ของฟีเจอร์เหล่านั้นถูกซ่อนไว้ ไม่รองรับ Wayland session

## รายการตรวจสอบสิทธิ์

### macOS

ก่อนเซสชันการจับภาพจริงครั้งแรก ตรวจสอบ:

1. `System Settings > Privacy & Security > Screen Recording`
2. `System Settings > Privacy & Security > Accessibility`

เปิดใช้งาน SnapTray ในทั้งสองที่ แล้วรีสตาร์ทแอปหากคุณเปลี่ยนสิทธิ์

### Windows

สิทธิ์รันไทม์ส่วนใหญ่จัดการโดย OS และรูปแบบการติดตั้ง หากคุณบันทึกเสียง ยืนยันว่าไมโครโฟนหรือแหล่งเสียงที่เลือกพร้อมใช้งานและได้รับอนุญาต

## ขั้นตอนที่ 1: ตรวจสอบการทำงานของถาดระบบ (1 นาที)

1. เปิด SnapTray
2. ยืนยันว่าไอคอนถาดระบบปรากฏขึ้น
3. เปิดเมนูและยืนยันว่ารายการเหล่านี้มีอยู่:
   - Region Capture
   - Screen Canvas
   - Settings

หากไอคอนถาดระบบไม่ปรากฏ ดำเนินการต่อด้วย [การแก้ไขปัญหา](/th/docs/troubleshooting/)

## ขั้นตอนที่ 2: ทดสอบการจับภาพหน้าจอ (2 นาที)

1. กด `F2`
2. ลากเลือกพื้นที่ใดก็ได้ และยืนยันว่าแว่นขยายและ overlay ขนาดปรากฏขึ้น
3. เพิ่มการมาร์กอัปลูกศรหนึ่งอัน
4. กด `Ctrl/Cmd + C` แล้ววางลงในโปรแกรมแก้ไขข้อความ
5. กด `Ctrl/Cmd + S` และยืนยันว่าไฟล์ถูกบันทึก

## ขั้นตอนที่ 3: ทดสอบการปักหมุด (2 นาที)

1. จับภาพอีกครั้งด้วย `F2`
2. กด `Enter` เพื่อปักหมุดผลลัพธ์
3. ย้าย ปรับขนาด และซูมหน้าต่างปักหมุด
4. สลับแอปและยืนยันว่าหมุดยังมองเห็นได้

## ขั้นตอนที่ 4: ตรวจสอบฟีเจอร์เฉพาะแพลตฟอร์ม (3 นาที)

### macOS และ Windows

1. เริ่มบันทึกจากเมนูถาดระบบหรือด้วย `snaptray record start`
2. บันทึก 5 ถึง 10 วินาที
3. หยุดและส่งออกเป็น MP4
4. ยืนยันเส้นทางเอาต์พุตใน Settings > Files

การบันทึกหน้าจอและ OCR ใช้ได้เฉพาะบน macOS และ Windows เท่านั้น

### Linux beta

ยืนยันว่าการดำเนินการ OCR และบันทึกไม่ปรากฏขึ้น แล้วดำเนินการต่อไปยังการตรวจสอบปุ่มลัด

## ขั้นตอนที่ 5: แก้ไขการชนกันของปุ่มลัด (2 นาที)

เปิด Settings > Hotkeys แล้วเปลี่ยนการกำหนดการดำเนินการที่ขัดแย้งกัน

การดำเนินการขั้นต่ำที่แนะนำ:

- Region Capture
- Screen Canvas
- Quick Pin
- Paste
