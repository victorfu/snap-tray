## 목표

이해 관계자가 빠르게 이해할 수 있는 짧고 명확한 화면 녹화를 제작합니다.

## 권장 길이 및 형식

- 이상적인 길이: 20~90초
- MP4: 대부분의 팀에 기본값
- GIF/WebP: 채팅과 문서를 위한 경량 미리보기

## 워크플로

### 1. 녹화 범위 정의

녹화는 macOS/Windows에서만 사용 가능합니다. Linux beta는 녹화를 포함하지 않으므로 해당 환경에서는 이 튜토리얼을 건너뛰세요.

- 워크스루와 데모에는 전체 화면 녹화를 사용합니다.
- 트레이 메뉴를 통해 시작하거나 `snaptray record start`를 실행합니다.

### 2. 녹화 전 설정 확인

설정 > 녹화에서 다음을 확인합니다.

- 프레임 레이트와 품질
- 카운트다운
- 오디오 소스 (마이크/시스템)

### 3. 의도적인 속도로 녹화

1. 녹화를 시작합니다.
2. 커서 움직임을 의도적으로 합니다.
3. 컨텍스트 전환 전에 필요하면 잠시 멈춥니다.
4. 요점이 증명되면 중지합니다.

### 4. 대상 청중에 맞게 내보내기

- 이슈 트래커와 프레젠테이션에는 MP4
- 빠른 비동기 업데이트에는 GIF/WebP

설명적인 파일명을 사용하세요.

`onboarding-flow-checkout-2026-02-24.mp4`

### 5. 컨텍스트 텍스트 첨부

다음 내용을 담은 한 단락을 포함합니다.

- 시청자가 관찰해야 할 내용
- 예상 결과
- 실제 결과 (버그 보고 시)

## 품질 체크리스트

- 녹화가 올바른 상태에서 시작됩니다 (설정 잡음 없음).
- 마우스 움직임과 동작을 따라가기 쉽습니다.
- 출력 형식이 대상 (트래커/채팅/문서)에 맞습니다.
- 파일명과 설명이 목적을 설명합니다.

## 계속 학습하기
