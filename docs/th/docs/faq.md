## SnapTray อัปโหลดภาพที่จับหรือไม่?

ไม่มีการอัปโหลดอัตโนมัติเป็นค่าเริ่มต้น การจับภาพ การมาร์กอัป และการปักหมุดอยู่ในเครื่องเว้นแต่คุณใช้การดำเนินการ Share URL อย่างชัดเจน การบันทึกเฉพาะ macOS/Windows และก็อยู่ในเครื่องเช่นกัน สำหรับบิลด์ดาวน์โหลดโดยตรง SnapTray ตรวจสอบการอัปเดตผ่าน updater เนทีฟของแพลตฟอร์ม (Sparkle บน macOS, WinSparkle บน Windows) ซึ่งสามารถปิดใช้งานได้ใน Settings

## ควรใช้รูปแบบเอาต์พุตใด?

การบันทึกและการส่งออกแบบเคลื่อนไหวเฉพาะ macOS/Windows บนแพลตฟอร์มเหล่านั้น:

- MP4 สำหรับวิดีโอยาวและบทช่วยสอน
- GIF สำหรับลูปสั้นหรือสาธิตเล็กน้อย
- WebP สำหรับคลิปเคลื่อนไหวที่เบากว่า

## ทำไม OCR ไม่สามารถใช้งานได้ในระบบของฉัน?

OCR ใช้ได้บน macOS/Windows เมื่อรองรับแพลตฟอร์มและแพ็กภาษาที่ติดตั้งอยู่ ซ่อนและไม่รวมอยู่ใน Linux beta

## สามารถกำหนดปุ่มลัดเริ่มต้นได้ไหม?

ได้ เปิด Settings > Hotkeys แล้วแก้ไขตามหมวดหมู่

## สามารถใช้ SnapTray จากสคริปต์ได้ไหม?

ได้ CLI คืออินเทอร์เฟซระบบอัตโนมัติอย่างเป็นทางการสำหรับการจับภาพในเครื่องและการควบคุมผ่าน IPC ดู [CLI](/th/docs/cli/) สำหรับข้อมูลอ้างอิงคำสั่งทั้งหมด

## ข้อจำกัดของแพลตฟอร์มปัจจุบันมีอะไรบ้าง?

- การจับภาพแบบหลายจอภาพและ mixed-DPI ยังได้ประโยชน์จากการทดสอบอุปกรณ์จริงมากขึ้น
- บน macOS/Windows การบันทึกหน้าจออาจถอยกลับไปใช้ Qt capture engine เมื่อ native API ไม่สามารถใช้ได้ ซึ่งอาจช้ากว่า
- การจับเสียงระบบบน macOS ต้องการ macOS 13+ หรืออุปกรณ์เสียงเสมือนเช่น BlackHole
- Linux beta เป็น Ubuntu 22.04 X11 AppImage; ไม่รวมการบันทึกและ OCR และไม่รองรับ Wayland session

## จะรายงานบักได้ที่ไหน?

เปิด issue บน GitHub และรวม:

- เวอร์ชัน OS
- เวอร์ชัน SnapTray
- ขั้นตอนการทำซ้ำ
- สกรีนช็อตหรือการบันทึกหากเกี่ยวข้อง
