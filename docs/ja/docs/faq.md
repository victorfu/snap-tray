## SnapTray はキャプチャをアップロードしますか？

自動アップロードは行われません。キャプチャ、注釈、ピン留めはローカルに保存されます。録画は macOS/Windows のみで、こちらもローカルに保存されます。ダイレクトダウンロードビルドでは、SnapTray はプラットフォームネイティブのアップデーター（macOS では Sparkle、Windows では WinSparkle）経由でアップデートを確認します。これは設定で無効にできます。

## どの出力フォーマットを使えばよいですか？

録画とアニメーションエクスポートは macOS/Windows のみです。それらのプラットフォームでは：

- 長い動画やチュートリアルには MP4
- 短いループやデモには GIF
- 軽量なアニメーションスニペットには WebP

## なぜ私のシステムで OCR が利用できないのですか？

OCR は、プラットフォームのサポートとインストールされた言語パックが存在する場合に macOS/Windows で利用可能です。Linux beta では非表示であり含まれていません。

## デフォルトのホットキーをカスタマイズできますか？

はい。設定 > ホットキー を開いてカテゴリ別に編集してください。

## SnapTray をスクリプトから使用できますか？

はい。CLI はローカルキャプチャと IPC ベースの操作のための公式自動化インターフェースです。完全なコマンドリファレンスは [CLI](/ja/docs/cli/) をご覧ください。

## 現在のプラットフォームの制限は何ですか？

- マルチモニターと混在 DPI のキャプチャはまだ実際のデバイスでのテストから恩恵を受けます
- macOS/Windows では、ネイティブ API が利用できない場合に画面録画が Qt キャプチャエンジンにフォールバックすることがあり、速度が低下する場合があります
- macOS でのシステム音声キャプチャには macOS 13+ または BlackHole などの仮想オーディオデバイスが必要です
- Linux beta は Ubuntu 22.04 X11 AppImage です；録画と OCR は含まれておらず、Wayland セッションはサポートされていません

## バグはどこに報告すればよいですか？

GitHub で Issue を開き、以下を含めてください：

- OS バージョン
- SnapTray バージョン
- 再現手順
- 関連するスクリーンショットや録画
