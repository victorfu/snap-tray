## 목표

엔지니어가 처음 시도에서 재현할 수 있는 버그 보고서를 작성합니다.

## 출력 패키지

- 주석이 달린 스크린샷 1장
- 간결한 재현 단계 목록 1개
- 선택 사항: 10~30초 녹화 (macOS/Windows 전용; Linux beta에서는 건너뜀)

## 워크플로

### 1. 한 번 재현하고 상태 고정

- 이슈를 재현합니다.
- 정확히 오류가 발생한 상태에서 멈춥니다.
- 예상 동작과 비교가 필요하다면 두 상태를 모두 캡처하고 고정합니다.

### 2. 증거 캡처

1. `F2`를 눌러 캡처합니다.
2. 주변 컨텍스트 (툴바, 탭 이름, 상태 영역)를 포함합니다.
3. 과도하게 자르지 마세요. 재현에 필요한 단서를 포함하세요.

### 3. 진단을 위한 주석

1. `단계 배지`를 사용하여 순서를 표시합니다 (`1`, `2`, `3`).
2. `화살표`로 오류 위치를 가리킵니다.
3. `텍스트`로 간결한 사실을 기록합니다: 오류 코드, 예상치 못한 값, 타임스탬프.
4. `모자이크`로 개인 토큰이나 사용자 데이터를 가립니다.

### 4. 내보내기 및 명확한 파일명 지정

다음과 같은 구조적인 이름으로 저장합니다.

`bug-login-timeout-win10-2026-02-24.png`

### 5. 선택 사항: 짧은 증거 영상 녹화

타이밍이나 애니메이션에 따른 이슈라면 macOS/Windows에서만 녹화합니다. Linux beta는 녹화를 포함하지 않으므로 해당 단계를 건너뜁니다.

1. 트레이 메뉴에서 전체 화면 녹화를 시작하거나 `snaptray record start`를 실행합니다.
2. 10~30초 녹화합니다.
3. 중지하고 MP4로 내보냅니다.

## 버그 보고서 템플릿

```markdown
제목: [플랫폼] 간결한 문제 요약

환경:
- 앱 버전:
- OS:

재현 단계:
1. ...
2. ...
3. ...

예상 동작:
실제 동작:

첨부 파일:
- screenshot.png
- recording.mp4 (선택 사항, macOS/Windows 전용)
```

## 품질 체크리스트

- 재현 단계가 결정론적입니다.
- 스크린샷에 문제와 컨텍스트가 모두 포함되어 있습니다.
- 주석이 순서와 영향을 설명합니다.
- 개인 데이터가 마스킹되어 있습니다.
