## 目標

ウィンドウを切り替えることなく、重要なビジュアル参照を常時表示して作業を速める。

## 最適なユースケース

- デザインモックからコーディング
- 期待する UI と実際の UI を比較
- 検証中にテスト手順を表示したままにする

## ワークフロー

### 1. 参照をキャプチャしてピン留め

1. `F2` で参照画像をキャプチャします。
2. `Enter` を押して結果を画面にピン留めします。

### 2. ピン留めウィンドウを快適に調整

- 必要な部分だけが見えるようにリサイズします。
- サブモニターの端や角に移動します。
- 見えるけれど邪魔にならないよう透明度を調整します。

### 3. 複雑なタスクには複数のピンを使用

以下のように別々のピンを作成します：

- デザインターゲット
- 現在の実装
- QA 受け入れ基準

### 4. ピン履歴を再利用

コンテキストが頻繁に変わる場合は、ピン履歴を開いて再キャプチャする代わりに以前の参照を復元します。

## 実用的なヒント

- テキスト要件用に小さなピンを 1 つ、ビジュアルの詳細用に大きなピンを 1 つ保持します。
- セットアップで利用可能な場合は、クイックピンホットキー（デフォルト：`Shift+F2`）を使用します。
- ピンが重要な UI に重なる場合は、移動する前に透明度を下げます。

## 品質チェックリスト

- タスク実行中も参照が常時表示されている。
- Alt+Tab の切り替え頻度が減っている。
- 履歴から以前の参照を復元できる。
