## 목표

창 전환 없이 중요한 시각적 참조를 항상 보이게 유지하여 더 빠르게 작업합니다.

## 최적 사용 사례

- 디자인 목업을 보며 코딩
- 예상 UI와 실제 UI 비교
- 검증 중 테스트 단계 항상 표시

## 워크플로

### 1. 참조 이미지 캡처 및 고정

1. `F2`로 참조 이미지를 캡처합니다.
2. `Enter`를 눌러 결과를 화면에 고정합니다.

### 2. 편안한 핀 창 설정

- 유용한 영역만 보이도록 크기를 조정합니다.
- 보조 모니터 가장자리나 모서리로 이동합니다.
- 방해되지 않으면서 잘 보이도록 불투명도를 조정합니다.

### 3. 복잡한 태스크에 여러 핀 사용

다음을 위해 별도의 핀을 만드세요.

- 디자인 목표
- 현재 구현
- QA 수용 기준

### 4. 핀 기록 재사용

컨텍스트가 자주 바뀌는 경우, 다시 캡처하는 대신 핀 기록을 열어 이전 참조를 복원하세요.

## 실용적인 팁

- 텍스트 요구 사항에는 작은 핀, 시각적 세부 사항에는 큰 핀을 유지하세요.
- 설정에서 사용 가능하다면 빠른 고정 단축키 (`Shift+F2` 기본값)를 사용하세요.
- 핀이 중요한 UI를 가리면 이동하기 전에 불투명도를 낮추세요.

## 품질 체크리스트

- 태스크 실행 내내 참조가 보입니다.
- Alt+Tab 전환 빈도가 줄어듭니다.
- 기록에서 이전 참조를 복원할 수 있습니다.
