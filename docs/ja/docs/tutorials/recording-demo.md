## 目標

ステークホルダーがすぐに理解できる短く明確な画面録画を作成する。

## 推奨時間とフォーマット

- 理想的な長さ：20〜90秒
- MP4：ほとんどのチームのデフォルト
- GIF/WebP：チャットやドキュメント用の軽量プレビュー

## ワークフロー

### 1. 録画スコープを定義

録画は macOS/Windows でのみ利用可能です。Linux beta には録画が含まれていないため、このチュートリアルはスキップしてください。

- ウォークスルーとデモにはフルスクリーン録画を使用します。
- トレイメニューから開始するか、`snaptray record start` を実行します。

### 2. 録画前に設定を確認

設定 > 録画 で以下を確認します：

- フレームレートと品質
- カウントダウン
- 音声ソース（マイク/システム）

### 3. 意図的なペースで録画

1. 録画を開始します。
2. カーソルの動きを意図的に保ちます。
3. コンテキストを切り替える前に必要に応じて一時停止します。
4. ポイントが証明できたら停止します。

### 4. 視聴者に合わせてエクスポート

- 課題トラッカーとプレゼンテーションには MP4
- クイックな非同期アップデートには GIF/WebP

説明的なファイル名を使用します：

`onboarding-flow-checkout-2026-02-24.mp4`

### 5. コンテキストテキストを添付

以下を含む 1 段落を記載します：

- 視聴者が観察すべき内容
- 期待する結果
- 実際の結果（バグを報告する場合）

## 品質チェックリスト

- 録画が正しい状態から始まっている（セットアップのノイズがない）。
- マウスの動きとアクションが追いやすい。
- 出力フォーマットが送信先に合っている（トラッカー/チャット/ドキュメント）。
- ファイル名とキャプションが目的を説明している。

## さらに学ぶ

[録画](/ja/docs/recording/) で録画オプションの詳細を確認し、[ホットキー](/ja/docs/hotkeys/) でショートカットを最適化してください。
