## 10분 안에 완료할 내용

- 권한 설정
- 첫 스크린샷 및 주석
- 첫 핀 창
- 플랫폼별 기능 확인
- 단축키 충돌 확인

## 실행 전제 조건

### macOS

- macOS 14 이상
- 화면 녹화 권한
- 창 감지 및 원활한 워크플로를 위해 접근성 권한 권장

### Windows

- Windows 10 이상
- 오디오 녹화 시에만 마이크 권한 필요

### Linux beta

SnapTray는 Ubuntu 22.04 X11 beta를 AppImage로 제공합니다. 녹화와 OCR은
Linux beta에 포함되지 않으며 해당 UI는 숨겨져 있습니다. Wayland 세션은
지원되지 않습니다.

## 권한 체크리스트

### macOS

첫 캡처 세션 전에 다음을 확인하세요.

1. `시스템 설정 > 개인 정보 보호 및 보안 > 화면 녹화`
2. `시스템 설정 > 개인 정보 보호 및 보안 > 접근성`

두 항목 모두에서 SnapTray를 활성화한 후, 권한을 변경했다면 앱을 재시작하세요.

### Windows

대부분의 런타임 권한은 OS와 설치 프로그램이 처리합니다. 오디오를 녹화하는 경우, 선택한 마이크 또는 오디오 소스가 사용 가능하고 허용되었는지 확인하세요.

## 1단계: 트레이 런타임 확인 (1분)

1. SnapTray를 실행합니다.
2. 트레이 아이콘이 표시되는지 확인합니다.
3. 메뉴를 열고 다음 항목이 있는지 확인합니다.
   - 영역 캡처
   - 스크린 캔버스
   - 설정

트레이 아이콘이 나타나지 않으면 [문제 해결](/ko/docs/troubleshooting/)을 참조하세요.

## 2단계: 스크린샷 흐름 테스트 (2분)

1. `F2`를 누릅니다.
2. 영역을 드래그하여 돋보기와 크기 오버레이가 나타나는지 확인합니다.
3. 화살표 주석을 하나 추가합니다.
4. `Ctrl/Cmd + C`를 눌러 텍스트 편집기에 붙여넣습니다.
5. `Ctrl/Cmd + S`를 눌러 파일이 저장되었는지 확인합니다.

## 3단계: 핀 워크플로 테스트 (2분)

1. `F2`로 다시 캡처합니다.
2. `Enter`를 눌러 결과를 고정합니다.
3. 핀 창을 이동, 크기 조정, 확대/축소합니다.
4. 다른 앱으로 전환하고 핀이 계속 보이는지 확인합니다.

## 4단계: 플랫폼별 기능 확인 (3분)

### macOS 및 Windows

1. 트레이 메뉴 또는 구성한 녹화 단축키로 녹화를 시작합니다.
2. 5~10초 녹화합니다.
3. 중지하고 MP4로 내보냅니다.
4. 설정 > 파일에서 출력 경로를 확인합니다.

녹화와 OCR은 macOS 및 Windows에서만 사용 가능합니다.

### Linux beta

OCR과 녹화 동작이 표시되지 않는지 확인한 후 단축키 확인으로 넘어가세요.

## 5단계: 단축키 충돌 해결 (2분)

설정 > 단축키를 열고 충돌하는 동작을 다시 바인딩하세요.

최소 권장 동작:

- 영역 캡처
- 스크린 캔버스
- 빠른 고정
- 붙여넣기
