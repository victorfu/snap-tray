## เป้าหมาย

ทำงานได้เร็วขึ้นโดยเก็บอ้างอิงภาพสำคัญให้มองเห็นได้ตลอดเวลาโดยไม่ต้องสลับหน้าต่าง

## กรณีการใช้งานที่ดีที่สุด

- เขียนโค้ดจาก design mock
- เปรียบเทียบ UI ที่คาดหวังกับ UI จริง
- เก็บขั้นตอนการทดสอบให้มองเห็นระหว่างการตรวจสอบ

## เวิร์กโฟลว์

### 1. จับภาพและปักหมุดอ้างอิง

1. จับภาพอ้างอิงด้วย `F2`
2. กด `Enter` เพื่อปักหมุดผลลัพธ์บนหน้าจอ

### 2. ปรับหน้าต่างปักหมุดเพื่อความสะดวก

- ปรับขนาดเพื่อเก็บเฉพาะพื้นที่ที่มีประโยชน์
- ย้ายไปที่ขอบหรือมุมของจอภาพรอง
- ปรับความโปร่งใสให้มองเห็นได้แต่ไม่บัง

### 3. ใช้หลายหมุดสำหรับงานซับซ้อน

สร้างหมุดแยกสำหรับ:

- เป้าหมายการออกแบบ
- การใช้งานปัจจุบัน
- เกณฑ์การยอมรับ QA

### 4. นำประวัติหมุดมาใช้ซ้ำ

เมื่อบริบทเปลี่ยนบ่อย เปิดประวัติหมุดและกู้คืนอ้างอิงก่อนหน้าแทนการจับภาพใหม่

## เคล็ดลับการใช้งาน

- เก็บหมุดเล็กสำหรับข้อกำหนดข้อความและหมุดใหญ่สำหรับรายละเอียดภาพ
- ใช้ปุ่มลัดปักหมุดด่วน (`Shift+F2` เป็นค่าเริ่มต้น) หากพร้อมใช้งานในการตั้งค่าของคุณ
- หากหมุดทับ UI สำคัญ ให้ลดความโปร่งใสก่อนย้าย

## รายการตรวจสอบคุณภาพ

- อ้างอิงมองเห็นได้ตลอดการทำงาน
- คุณลด alt-tab ได้
- คุณสามารถกู้คืนอ้างอิงก่อนหน้าจากประวัติได้

## บทช่วยสอนถัดไป

เมื่อต้องการอธิบายพฤติกรรมแบบไดนามิกบน macOS หรือ Windows ดำเนินการต่อด้วย [สาธิตการบันทึก (เฉพาะ macOS/Windows)](/th/docs/tutorials/recording-demo/) ผู้ใช้ Linux beta สามารถข้ามบทช่วยสอนการบันทึกได้
