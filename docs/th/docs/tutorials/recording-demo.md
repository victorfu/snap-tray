## เป้าหมาย

สร้างการบันทึกหน้าจอสั้นและชัดเจนที่ผู้มีส่วนได้ส่วนเสียสามารถเข้าใจได้อย่างรวดเร็ว

## ระยะเวลาและรูปแบบที่แนะนำ

- ความยาวที่เหมาะสม: 20-90 วินาที
- MP4: ค่าเริ่มต้นสำหรับทีมส่วนใหญ่
- GIF/WebP: ตัวอย่างขนาดเบาสำหรับ chat และเอกสาร

## เวิร์กโฟลว์

### 1. กำหนดขอบเขตการบันทึก

การบันทึกใช้ได้เฉพาะบน macOS/Windows เท่านั้น Linux beta ไม่รวมการบันทึก ให้ข้ามบทช่วยสอนนี้บนแพลตฟอร์มนั้น

- ใช้การบันทึกเต็มหน้าจอสำหรับ walkthrough และการสาธิต
- เริ่มผ่านเมนูถาดระบบหรือรัน `snaptray record start`

### 2. กำหนดค่าก่อนบันทึก

ใน Settings > Recording ยืนยัน:

- เฟรมเรตและคุณภาพ
- การนับถอยหลัง
- แหล่งเสียง (ไมค์/ระบบ)

### 3. บันทึกอย่างมีจังหวะ

1. เริ่มบันทึก
2. เคลื่อนเคอร์เซอร์อย่างมีเจตนา
3. หยุดชั่วคราวหากจำเป็นก่อนเปลี่ยนบริบท
4. หยุดเมื่อพิสูจน์ประเด็นได้แล้ว

### 4. ส่งออกสำหรับกลุ่มเป้าหมาย

- MP4 สำหรับ issue tracker และการนำเสนอ
- GIF/WebP สำหรับอัปเดตแบบ async อย่างรวดเร็ว

ใช้ชื่อไฟล์ที่สื่อความหมาย:

`onboarding-flow-checkout-2026-02-24.mp4`

### 5. แนบข้อความบริบท

รวมย่อหนึ่งย่อหน้าที่มี:

- สิ่งที่ผู้ชมควรสังเกต
- ผลลัพธ์ที่คาดหวัง
- ผลลัพธ์จริง (หากรายงานบัก)

## รายการตรวจสอบคุณภาพ

- การบันทึกเริ่มในสถานะที่ถูกต้อง (ไม่มีเสียงรบกวนจากการตั้งค่า)
- การเคลื่อนเมาส์และการดำเนินการติดตามได้ง่าย
- รูปแบบเอาต์พุตตรงกับปลายทาง (tracker/chat/เอกสาร)
- ชื่อไฟล์และคำบรรยายอธิบายวัตถุประสงค์

## เรียนรู้ต่อ
